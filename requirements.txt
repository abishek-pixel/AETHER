# ── Embeddings ────────────────────────────────────────────────
# sentence-transformers 2.7.x is the last generation that does NOT use
# torch directly inside encode() at call-time.  Version 3+ / 5+ decorates
# encode() with @torch.inference_mode() which causes:
#     NameError: name 'torch' is not defined
# when torch is absent from the environment — even if the import succeeded.
#
# On Render (512 MB RAM, no GPU), torch is a soft dependency:
#   - If Render resolves the CPU wheel correctly, full embeddings work.
#   - If torch is absent, EmbeddingService falls back to zero-vector
#     embeddings and the research pipeline continues without ML features.
#
# CPU-only torch wheel index (Render may or may not resolve this):
